Moduli del ciclo di vita

Utilizzare la pagina Moduli del ciclo di vita per configurare i moduli del ciclo di vita.

I moduli del ciclo di vita eseguono task specifici quando vengono attivati da uno o più eventi nel ciclo di vita del server. Gli eventi trigger del server possibili sono: inizializzazione, avvio, preparazione all'elaborazione delle richieste e chiusura. I moduli del ciclo di vita non fanno parte della specifica Java EE, ma costituiscono un miglioramento apportato a GlassFish Server.

Per ogni modulo del ciclo di vita vengono fornite le informazioni riportate di seguito.

Nome

Il nome del modulo del ciclo di vita.

Abilitato o Stato

Il segno di spunta indica che l'applicazione è abilitata, mentre la X indica che è disabilitata, se esiste solo l'istanza server predefinita server.

Se esistono cluster o altre istanze server standalone, selezionare Abilitato in tutte le destinazioni per visualizzare le destinazioni sulle quali l'applicazione è distribuita.

Ordine di caricamento

Questo valore specifica l'ordine relativo in cui si desidera caricare il modulo del ciclo di vita.

La tabella Moduli del ciclo di vita contiene inoltre le opzioni riportate di seguito.

Nuovo

Pulsante che consente di creare un nuovo modulo del ciclo di vita.

Elimina

Pulsante che consente di eliminare uno o più moduli del ciclo di vita selezionati.

Abilita

Pulsante che consente di abilitare uno o più moduli del ciclo di vita selezionati. Presente se esiste solo l'istanza server predefinita server.

Disabilita

Pulsante che consente di disabilitare uno o più moduli del ciclo di vita selezionati. Presente se esiste solo l'istanza server predefinita server.
