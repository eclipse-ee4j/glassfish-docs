Noms JNDI logiques

Lorsque vous créez une ressource, vous indiquez un nom JNDI unique pour la ressource. Les applications utilisent ce nom pour accéder à la ressource.

La norme Java EE spécifie que certaines ressources par défaut doivent être rendues disponibles pour les applications et définit des noms JNDI spécifiques pour ces ressources par défaut. GlassFish Server rend ces noms disponibles via l'utilisation de noms JNDI logiques, qui mettent en correspondance les noms JDNI standard Java EE avec des ressources GlassFish Server spécifiques comme suit :

java:comp/DefaultContextService

Ce nom standard Java EE est mis en correspondance avec la ressource concurrent/__defaultContextService.

java:comp/DefaultManagedThreadFactory

Ce nom standard Java EE est mis en correspondance avec la ressource concurrent/__defaultManagedThreadFactory.

java:comp/DefaultManagedExecutorService

Ce nom standard Java EE est mis en correspondance avec la ressource concurrent/__defaultManagedExecutorService.

java:comp/DefaultManagedScheduledExecutorService

Ce nom standard Java EE est mis en correspondance avec la ressource concurrent/__defaultManagedScheduledExecutorService.

java:comp/DefaultDataSource

Ce nom standard Java EE est mis en correspondance avec la ressource jdbc/__default.

java:comp/DefaultJMSConnectionFactory

Ce nom standard Java EE est mis en correspondance avec la ressource jms/__defaultConnectionFactory.
